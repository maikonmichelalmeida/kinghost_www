Curso Concluído

Este curso agora cobre o caminho completo de verificação introdutória: por que verificar, como planejar, como medir cobertura, por que linguagens/metodologias de verificação existem, e como VCS/Verdi entram no fluxo prático. A próxima aplicação natural é transformar a simulação do CPU em regressão com testes nomeados, PASS/FAIL automático, FSDB para debug e cobertura inicial.

Relação com o projeto

Este curso é a ponte entre “rodar uma simulação” e montar um ambiente de teste reutilizável. Para o laboratório do professor, ele prepara o vocabulário de driver, monitor, scoreboard, checker, coverage, regressão e debug com waveform. Isso conversa diretamente com os Makefiles de simulação e com a ideia de reaproveitar o ambiente para testar outros blocos.
